-------------------mysql备忘----------------
Linux下的mysql默认是要区分数据库名，表名，表别名大小写的；列名，列别名不区分大小写。
而在windows下全部不区分大小写。
show Variables like '%table_names'    --- 0--表示区分  1--不区分
mysql对于类型为varchar数据默认不区分大小写，但如果该字段以“*_bin”编码，会使mysql对其区分大小写。
utf8_general_ci    ---不区分大小写
utf8_bin           ---区分大小写

---清查询缓存
reset query cache;
---不缓存查询某语句
select sql_no_cache   * from test ...
---强制缓存sql_cache
select sql_cache * from table_name;
---设置客户端连接字符集
set names utf8;
---执行查询计划
explain select ...
---强制使用索引
select * from tablea force index(aa_index)
select * from tba force index(aa_index)
join tbb force index(bb_index) on tba.id=tab.aid
---强制连接顺序straight_join
select a.f1,b.f2 from a straight_join b where b.aid=a.id
上面的sql语句可知，通过straight_join强迫mysql按a、b的顺序连接表。

window下用source还原数据库中文乱码问题无法回避,因为window下的cmd命令字符集固定的gbk,不能换成其他字符集,
变通的做法直接把sql语句在sql客户端客户端执行,而不用cmd命令行去操作

mysql中,float占4个字节，double占8个字节，decimal(M,D)占M+2个字节。
decimal(5, 2) 的最大值为9999.99，因为有7 个字节可用, decimal(p,s)中p最大为65，S最大为30

在多表关联查询中，类似如下
select a.*,b.* from a left join b on a.id=b.aid and b.aa=?
where a.cc=? and b.kk=?
对于左外连接查询，如果右表b有非空记录条件刷选,应当把条件放到 on.... and 后面，这样可以对b进行条件刷选后去关联a.
如果把此条件放到where下面，左外连接就变成join内连接了。

select a.*,b.* from t_a a
left join t_b b on a.id=b.a_id
where b.a_id is null
这样查出的数据是a表在b表中没关联数据的记录
